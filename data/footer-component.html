Footer component

X-Framework provides footer as a component.

The following snippet will insert a footer: 

<div data-component="FooterComponent" data-id="MY-COMPONENT-ID"></div>

To use one global footer for the whole application insert the snippet as a sibling of all the .xf-page elements: 

<body>
	<div class="xf-page" id="page1">...</div>
	<div class="xf-page" id="page2">...</div>
    <div data-component="FooterComponent" data-id="MY-COMPONENT-ID"></div>
</body>

It is also possible to put headers inside the necessary pages:

<body>
	<div class="xf-page" id="page1">
		<div data-component="FooterComponent" data-id="MY-COMPONENT-ID"></div>
	</div>
	<div class="xf-page" id="page2">
		<div data-component="FooterComponent" data-id="MY-SECOND-COMPONENT-ID"></div>
	</div>
</body>

Note that it is not possible to have a single non-fixed footer due to the fact that the pages are positioned absolutely and cannot account for the footer being present.

The Footer component has several configurable properties. They can be set using XF.setOptionsByID method. Here are the default values:

XF.setOptionsByID(
    'COMPONENT-ID',
    {
        isFixed : true,
        buttonsClass : '',
        buttons : [],
    }
);
isFixed: true
Indicates whether the footer has to be fixed at the bottom of the screen or scroll together with the page
buttonsClass: null,
custom class name to be applied on EACH button
buttons: [],
an array of objects describing buttons. Each button has the following properties:
buttonClass: null
custom class to be applied on button
isBackBtn: true
true | false
text: null
text to be displayed
textClass: null
custom class name to be applied on text
icon: null
icon name. See Icons section for full list of icons
iconClass: null
custom class name for the icon
href: null
URL to navigate to when clicked

If the component does not have the features your project requires, feel free to copy the component definition and template files and modify them to suit your needs.
