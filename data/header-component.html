Header component

X-Framework provides Header as a component.

The following snippet will insert a header: 

<div data-component="HeaderComponent" data-id="MY-COMPONENT-ID"></div>

To use one global header for the whole application insert the snippet as a sibling of all the .xf-page elements: 

<body>
	<div data-component="HeaderComponent" data-id="MY-COMPONENT-ID"></div>
	<div class="xf-page" id="page1">...</div>
	<div class="xf-page" id="page2">...</div>
</body>

It is also possible to put headers inside the necessary pages:

<body>
	<div class="xf-page" id="page1">
		<div data-component="HeaderComponent" data-id="MY-COMPONENT-ID"></div>
	</div>
	<div class="xf-page" id="page2">
		<div data-component="HeaderComponent" data-id="MY-SECOND-COMPONENT-ID"></div>
	</div>
</body>

Note that it is not possible to have a single non-fixed header due to the fact that the pages are positioned absolutely and cannot account for the header being present.

The Header component has several configurable properties. They can be set using XF.setOptionsByID method. Here are the default values:

XF.setOptionsByID(
    'COMPONENT-ID',
  {
  headerElement: null,
  isFixed: true,
  buttonsClass: null,
  title: 'Page Title',
  buttons: [{
            align: 'left'
            buttonClass: null,
            isBackBtn: true,
            isSpecial: false,
            text: null,
            textClass: null,
            icon: null,
            iconClass: null,
            href: null,
            tooltip: 'Go to previous page',
            params: {
                data: 'back'
            }
        }]
    }
);
isFixed: true
adding 'xf-header-fixed' class
buttonsClass: null,
custom class name to be applied on EACH button
title: 'Page Title',
header title text
headerElement: null
element in which title will be wrapped
buttons
an array of objects describing buttons, each can have the following properties:
align: 'left'
position: 'left' | 'right'
buttonClass: null
custom class to be applied on button
isBackBtn: true
true | false
isSpecial: false
true | false - changes the view only
text: null
text to be displayed
textClass: null
custom class name to be applied on text
icon: null
icon name. See Icons section for full list of icons
iconClass: null
custom class name for the icon
href: null
URL to navigate to when clicked
tooltip:'Go to previous page'
button's title attribute
params: {data: 'back'}
params to be passed into the dispatched event ('buttonClick')

If the component does not have the features your project requires, feel free to copy the component definition and template files and modify them tou suit your needs.
