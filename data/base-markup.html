Below is a sample of a basic page structure needed for X-Framework: 

<!DOCTYPE html>
<html>
<head>
	<title>My App Title</title>
	<meta name="viewport" content="width=device-width, initial-scale=1">
	<link rel="stylesheet" href="../styles/xf.css"/>
	<script src="../js/xf.js"></script>
</head>
<body>

<div class="xf-page" id="page1-id">
	<div class="xf-header">
		<h1>My Title</h1>
	</div>

	<div class="xf-page-content">
		<p>Hello world</p>
	</div>
</div>
</body>
</html>

Note that there is a meta viewport tag in the head to specify how the browser should display the page zoom level and dimensions. If this isn't set, many mobile browsers will use a "virtual" page width around 800-900 pixels to make it work well with existing desktop sites but mobile screens may look zoomed out and too wide.

Navigation between screens is performed by changing containers with .xf-page class.
